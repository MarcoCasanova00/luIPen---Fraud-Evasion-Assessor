About This Tool

Purpose

The Fraud Evasion Penetration Testing Tool is designed for corporate security teams and penetration testers to assess the resilience of fraud detection systems against various attack vectors.

This is a DEFENSIVE security tool that helps organizations understand and improve their fraud detection capabilities.

Features

  • Detection Resilience Scoring - Evaluates how well transaction patterns might bypass fraud detection
  • Vulnerability Reports - Provides actionable recommendations for improving fraud controls
  • GeoIP Integration - Optional MaxMind GeoLite2 support for IP geolocation
  • Educational Context - Explains fraud detection concepts and best practices

Ethical Use

This tool must only be used:

  • On systems you own or have explicit written permission to test
  • For authorized security assessments and educational purposes
  • In compliance with applicable laws and regulations

Unauthorized use is strictly prohibited.

API Documentation

Available Endpoints:

GET  /health           - Health check
GET  /                 - Main interface

POST /api/score        - Calculate detection resilience score
    Request: { target_url, distance_km, ratio_to_median_purchase_price,
               retailer_history, has_chip, has_pin, is_online }
    Response: { success_score, fraud_probability, is_bypassed, confidence }

POST /api/report       - Generate vulnerability report
    Request: Same as /api/score
    Response: { report_id, findings, recommendations, risk_profile }

POST /api/geoip/distance - Calculate IP to billing address distance
    Request: { ip_address, billing_lat, billing_lon }
    Response: { distance_km, ip_location }

GET  /api/demo-presets - Get available test presets
